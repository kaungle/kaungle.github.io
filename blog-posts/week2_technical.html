Displaying Inline vs. Inline Blocks


"What Are You Talking About?"

When stylizing webpages in CSS-land, developers have the option of using code to 'display'content -- text, images, etc. -- in different ways. The most direct way to go about this, as you can imagine, is by simply typing 'display' and a specific value into your stylesheet. Now, this is where 'inline' & 'inline blocks' come into the equation, as two of the more commong variables assigned to the 'display' property.


"What do you mean you 'spreadsheet?'

That is just a fancy name for the file we use to decorate web pages. In this case, we might add some code that looks something like this: ul.products li { width: 200px; display: inline-block; } to order our program to display the list values contained within the products unordered list class.

Inline Block?

When CSS describes how to display elements, 'inline' refers to those that take up only the space they need and can share their horizontal line with other elements. 'Block' elements, on the other hand, take up the full width of their parent elements and have their own horizontal line. Combining these terms, 'inline-block', makes an element sit inline while still having block-level children elements (or elements within elements) that can take up multiple lines.
